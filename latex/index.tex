\hypertarget{index_autotoc_md3}{}\doxysection{picotls}\label{index_autotoc_md3}
Picotls is a \href{https://tools.ietf.org/html/rfc8446}{\texttt{ TLS 1.\+3 (RFC 8446)}} protocol stack written in C, with the following features\+:
\begin{DoxyItemize}
\item support for three crypto engines
\begin{DoxyItemize}
\item \char`\"{}\+Open\+SSL\char`\"{} backend using libcrypto for crypto and X.\+509 operations
\item \char`\"{}minicrypto\char`\"{} backend using \href{https://github.com/ctz/cifra}{\texttt{ cifra}} for most crypto and \href{https://github.com/kmackay/micro-ecc}{\texttt{ micro-\/ecc}} for secp256r1
\item \href{https://github.com/h2o/picotls/pull/310}{\texttt{ "{}fusion"{} AES-\/\+GCM engine, optimized for QUIC and other protocols that use short AEAD blocks}}
\end{DoxyItemize}
\item support for PSK, PSK-\/\+DHE resumption using 0-\/RTT
\item API for dealing directly with TLS handshake messages (essential for QUIC)
\item supported extensions\+:
\begin{DoxyItemize}
\item RFC 7250 (raw public keys)
\item RFC 8879 (certificate compression)
\item Encrypted SNI (wg-\/draft-\/02)
\end{DoxyItemize}
\end{DoxyItemize}

Primary goal of the project is to create a fast, tiny, low-\/latency TLS 1.\+3 implementation that can be used with the HTTP/2 protocol stack and the upcoming QUIC stack of the \href{https://h2o.examp1e.net}{\texttt{ H2O HTTP/2 server}}.

The TLS protocol implementation of picotls is licensed under the MIT license.

Using resumption\+:

\DoxyCodeLine{\% ./cli -\/s session-\/file 127.0.0.1 8443}

\end{DoxyCode}
 The session-\/file is read-\/write. The cli server implements a single-\/entry session cache. The cli server sends New\+Session\+Ticket when it first sends application data after receiving Client\+Finished.

Using early-\/data\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\% ./cli -\/s session-\/file -\/e 127.0.0.1 8443}

\end{DoxyCode}
